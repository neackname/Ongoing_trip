# 前端设计文档（微信小程序）

本文档描述“随心游”小程序前端的技术选型、工程结构、关键模块设计与 UI 风格规范，便于答辩展示与后续维护。

## 1. 技术选型

- 平台：微信小程序（WXML/WXSS/JavaScript）
- 地图能力：
  - map 组件（marker/polyline/include-points/show-location）
  - 腾讯位置服务 SDK：`qqmap-wx-jssdk`（地点搜索/景点搜索/路线规划 direction）
- 网络请求：
  - `wx.request`
  - 统一封装：`common/utils/util.js`
- 登录鉴权：
  - `wx.login` 获取 code
  - 后端 `/travel/login` 换取 token
  - token 存储：`wx.setStorageSync('token', token)`
  - 请求头：`Authorization: Bearer <token>`（由 util.request 自动注入）

## 2. 工程结构

项目采用“主包 + 分包”组织页面，降低首包体积、便于模块化维护。

- 主包：
  - `pages/home`：首页（搜索、公告、文章推荐）
  - `pages/map`：定制页（地图、搜索、推荐、路线规划、足迹保存）
  - `pages/user`：我的（登录、个人入口）
  - `common/utils`：请求封装、token 管理
  - `libs`：第三方 SDK（qqmap）
- 分包：
  - `packages/home`：文章列表、文章详情、文章相关页面
  - `packages/user`：收藏、个人足迹、我的文章、资料修改等

## 3. 网络与鉴权设计

### 3.1 统一请求封装

`common/utils/util.js` 对外提供：
- `ensureToken()`：保证 token 可用（必要时触发登录）
- `request({ url, method, data, header, auth })`：统一请求入口，默认 `auth=true` 自动带 token

设计目标：
- 页面不关心 token 细节，只关心业务接口
- 错误处理集中化，减少重复代码

### 3.2 登录策略

登录有两种使用方式：
- 显式登录：用户在“我的”页点击登录按钮触发 `wx.login` → `/travel/login`
- 自动鉴权：页面调用 `util.request` 时若无 token，`ensureToken` 会自动登录补齐

项目可按答辩场景选择其中一种或两者并存。

## 4. 地图与定制页设计

### 4.1 顶部地点搜索（高德简洁风）

- 搜索框统一为白底圆角、轻阴影、右侧蓝色“搜索”
- 点击搜索后请求地点联想结果并显示列表
- 点击结果后在地图中：
  - 标注“当前位置”与“目的地”
  - 自动调整视野包含关键点（includePoints/includePoints API）

### 4.2 路线规划与渲染

- 支持模式：
  - 驾车 driving
  - 公交 transit
  - 步行 walking
- 路线规划返回 polyline 后解码并绘制
- 同时展示“预计时间/距离”（从路线返回提取 duration/distance）

### 4.3 足迹保存

保存内容包含：
- 起点/终点经纬度（后台存储与复现路线使用）
- 起点/终点名称（前端展示使用）
- 路线模式（driving/transit/walking）
- 路线规划原始结果（JSON）

交互：
- 地图右下角提供浮动“保存”按钮
- 当底部面板上滑覆盖地图时按钮隐藏，避免遮挡

## 5. UI 设计规范（高德简洁风）

统一风格目标：干净、留白、信息层级清晰、少颜色。

- 背景色：浅灰 `#f7f8fa`
- 内容卡片：白底、圆角、轻阴影
- 主色：蓝色 `#1a6cff`（操作按钮/高亮）
- 字号层级：
  - 标题：32rpx
  - 正文：28–30rpx
  - 辅助信息：24rpx
- 交互控件：
  - 搜索框统一样式（首页与定制页一致）
  - 列表项提供明确点击反馈（toast/modal/actionSheet）

## 6. 关键页面交互说明

- 首页：
  - 顶部搜索可切换“文章/用户”模式
  - 文章推荐最多 5 篇，点击进入文章详情
- 文章详情：
  - 收藏、点赞、评论发布与删除
  - 点作者可查看作者文章列表
- 我的：
  - 文章收藏、足迹收藏、个人足迹、我的文章入口
  - 个人足迹支持删除（仅删除自己创建的足迹）
